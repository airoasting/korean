# 문체 가이드 (윤문 심화)

korean(윤문) 스킬이 참조하는 한국어 문체·금지 사항·마무리 문장 심화 가이드. `patterns.md`의 14패턴 중 11~14번(어색한 동사구·AI 마무리 명언·식상한 비유·인용 동사 generic화)이 여기 더 자세히 정리돼 있다. 이 파일은 스킬 안에 번들돼 있어 외부 공유 경로에 의존하지 않는다.

## 목차 (필요한 절만 골라 읽는다)

- **§1 한국어 문체 기본**: 주술 호응, 번역체, 종결 통일, 숫자·시제
- **§2 인용·발화 동사**: 구체적 술어 표 (밝혔다·경고했다·반박했다)
- **§3 어색한 동사구**: 번역체 변종 교정 표
- **§4 식상한 비유 (절대 금지)**: 법정에 섰다·포문을 열었다 등
- **§5 AI 마무리 명언 (절대 금지)**: 시대가 왔습니다·신호입니다 등
- **§6 마무리 문장 가이드**: 사실로 닫는 4가지 자료
- **§7 제목 5원칙**: 진짜 행위자·전문어·구체 동사·날짜 금지
- **§8 직접 인용 번역**: 영어 구문 베끼지 말고 한국어 발화로 재현
- **§9 의미 정확성 검수**: 주어·목적어, 중복, 접속사, 문장 연결
- **§10 외래 조직·기술 용어 처리**: pod·squad 영문 병기 + 설명
- **§11 숫자와 비율 함께 표기**: 큰 수치는 분모와 함께
- **§12 자동 fail 그룹 (절대 금지)**: em dash·이중 피동·추측 어미·메타 표현
- **§13 일한 번역투 (일본어 직역)**: ~에 다름 아니다·에 있어서·이중부정·낫표·~的
- **§14 담화 층위**: 응결 장치 과잉·주제 전개·문장부호로 받치는 흐름

## 1. 한국어 문체 기본

1. **주술 구조 맞추기.** 한국어 어순으로. 주어 분명, 동사로 끝남.
2. **번역체 거르기.** "~에 의해 → ~가", "~을 가지고 있다 → ~이 있다", "~에 대한 → ~의", "그것은 ~이다 → 주어 살려 다시 쓰기".
3. **어미는 "~습니다"로 통일.** 단정적이고 정중하게. rhetorical 어미(~느냐, ~인가) 금지.
4. **숫자는 한글 단위.** "10퍼센트", "1,150억 달러", "8,000명". 날짜는 "5월 20일".
5. **짧은 문장과 긴 문장 섞기.** 사실 나열은 길게, 의미 부여는 짧게.
6. **단정형.** "~할 것 같습니다 → ~합니다 / ~할 것입니다". 추측 절제.
7. **인접 두 문장에서 같은 핵심 단어 반복 피하기.** 한 번을 풀어쓴다.
8. **시제 정확히.** 미래는 미래로. 옛 표기 금지 ("씌어 → 쓰여", "되어졌다 → 됐다").
9. **명사 중심을 동사 중심으로.** 영어는 명사를 즐기고 한국어는 동사를 즐긴다(이희재 2009, 김정우 2012). 영어 명사구를 그대로 옮긴 자리는 동사로 푼다. "독일의 국제연맹 탈퇴는 관계 증진을 위한 상호 관심을 증강시켰다" → "독일이 국제연맹에서 탈퇴하면서 두 나라 모두 관계 증진에 더 관심을 쏟았다". 특히 무생물 주어 + `~시켰다/~을 가져왔다`가 신호.
10. **잉여 대명사 덜어내기.** 영어는 주어·목적어 대명사가 의무지만 한국어는 자주 생략한다(김정우 2012, 이희재 2009). "그들은 그것을 분석했고, 그것이 유의미하다고 판단했다" → "분석해 보니 유의미했다". `그것·그들·그녀`가 잦으면 영형(생략)이나 명사로 되받는다.

## 2. 인용·발화 동사: 구체적으로

"~다고 했습니다"는 행위가 안 잡히는 편의적 술어. 발화 성격에 맞춰 구체화.

| 발화 성격 | 동사 |
|---|---|
| 공식 공지 | **밝혔다 / 발표했다 / 공시했다** |
| 경계 발언 | **경고했다 / 우려를 표했다** |
| 핵심 메시지 | **강조했다 / 시사했다** |
| 입장 표명 | **주장했다** |
| 대응 발언 | **반박했다 / 부인했다** |
| 응답 | **답했다 / 답변했다** |
| 추정 | **전망했다 / 예상했다 / 추정했다** |
| 매체 보도 | **보도됐다 / 알려졌다** |

원칙: 발화의 성격(공식 발표 / 경고 / 반박 / 응답 / 추정 / 보도)이 술어에 보여야 한다.

## 3. 어색한 동사구 (번역체 변종)

추상에 쓰이는 부자연스러운 동사구. 자연스러운 자리로.

| ❌ 어색 | ✅ 자연스러움 |
|---|---|
| X 달러를 향했습니다 | X 달러에 달했습니다 / X 달러로 늘었습니다 |
| X로 흘러갔습니다 | X에 가까워졌습니다 / X 쪽으로 기울었습니다 |
| X에 다가가고 있습니다 | X에 거의 도달했습니다 |
| X를 마주했습니다 (face 직역) | X을 만났습니다 / X에 맞닥뜨렸습니다 |
| X를 그렸습니다 (추상에) | X을 상정했습니다 / X로 가정했습니다 |
| X을 풀어냅니다 (남용) | X을 처리합니다 / 풀어냅니다 (1~2회만) |
| X를 부과하기 시작했습니다 (begin to 직역) | X했습니다 / X를 제한했습니다 (직접 동사로) |
| A가 B를 C로 만들었습니다 (made B into C 사역 직역) | A가 ~했고 B는 C가 됐습니다 (결과를 B 주어로) |

## 4. 식상한 비유: 절대 금지

한국 매체에서 너무 흔히 써서 헤드라인이 평범해진다. AI도 자주 만들어내는 클리셰.

**즉시 교정 대상**:
- 법정에 섰습니다 / 법정 공방에 들어섰습니다
- 본진이 나섰다 / 본진이 ~을 시작했다
- 수면 위로 떠올랐습니다
- 발길을 돌렸습니다
- 도전장을 내밀었습니다
- 포문을 열었습니다
- 닻을 올렸습니다
- 기지개를 켰습니다
- 춘추전국시대가 열렸습니다
- 한 발짝 다가섰습니다
- 신호탄을 쏘아 올렸습니다

**약한 클리셰**:
- 시동을 걸었다 / 엔진을 가동했다
- 본격 가속 / 급물살 / 드라이브

**대체**: 사건·인물·숫자의 구체로 풀어 쓴다. 비유 굳이 쓸 거면 본문 사실에서만 끌어올린 신선한 것으로.

## 5. AI 마무리 명언 (가짜 명언): 절대 금지

본문 사실에서 한 단계 추상으로 점프하는 마무리. LLM이 자주 만들어내는 함정.

**즉시 교정 대상 (마무리 자리에 두지 않는다)**:
- 자기 길을 찾는 중입니다
- 시대가 왔습니다 / 새로운 시대가 열렸습니다
- 시간이 시작됐습니다
- 신호입니다 / 신호로 읽힙니다
- 선이 그어지는 자리입니다
- 기로에 섰습니다 / 중대한 분기점입니다
- 셈입니다 + 추상 일반화 (예: "한 발 후퇴한 셈입니다")
- 역사가 어떻게 평가할지 지켜볼 일입니다
- 그림이 분명해졌습니다 / 풍경이 분명해졌습니다
- 베팅이 시장에 등장했습니다 / 분위기가 바뀌었습니다
- 마지막 역설은 여기에 있습니다 / 여기서 아이러니는 ~입니다 (역설·아이러니를 이름 붙여 예고)

**판정 기준**: 본문에 깔린 사실보다 한 단계 더 추상화된 인생론·당위·통찰이면 가짜 명언.

**예외**: 사실의 직접 압축은 OK. 메타 8,000명 감원 예시의 "줄어드는 건 결국 사람입니다"는 직전 사실(7,250억 vs 1,580억 매출 격차)에서 직접 나온 단언이라 살아남는다.

## 6. 마무리 문장 가이드: 사실로 닫는다

직전까지 사실 깔아 놓고 마지막 줄에서 "의미 정리"하면 AI 티의 결정타. **추상화·일반화·해석은 끝자리에 두지 않는다.**

기준 예시 (메타 감원 문단의 실제 마지막 문장):
> "마이크로소프트도 같은 주에 직원 7퍼센트에게 자발적 퇴직을 제안했습니다."

마지막 문장으로 쓸 수 있는 자료 4가지:

1. **같은 주 다른 사건**: 다른 회사·다른 나라가 같은 시기에 한 일
   - "같은 날 워싱턴에서는 백악관이 정반대 방향의 행정명령을 작성 중이라고 했습니다."

2. **역사적 대조**: 1년 전·5년 전 같은 시점 수치
   - "1년 전엔 5퍼센트 미만이었습니다."

3. **본문 인물·회사의 다른 사실**: 본문 사건과 직접 연결되는 별개 사실
   - "투로우는 1980~90년대 베스트셀러 법정 스릴러 작가입니다."

4. **다음 일정·후속 사건**: 본문 사건이 이어지는 시점
   - "정식 채택은 6월 둘째 주로 예정돼 있습니다."

**위트는 본문 안에 한 줄, 끝자리에는 두지 않는다.** 본문 중간에 punchy 라인 1줄 → 그 뒤 사실 한 줄로 닫기.

**교차 인용은 한 번만.** A·B 사건이 직접 연관되더라도 두 항목 모두에서 서로 cross-ref하면 메아리. 무게 가벼운 쪽 또는 시간상 뒤 항목만 cross-ref.

**역설·아이러니를 이름 붙여 예고하지 않는다.** "마지막 역설은 여기에 있습니다", "여기서 아이러니는" 같은 메타 예고는 역설을 김빠지게 한다. 상반된 사실을 나란히 놓아 독자가 스스로 느끼게 한다 (보여주고 말하지 않기). 콜백(첫 문장의 장면·소재를 마지막에 다시 부르기)은 여운을 만드는 좋은 마무리 도구다.

## 7. 제목 5원칙

**(1) 주어는 진짜 행위자.** 도구·제품·모델이 사건을 일으켰으면 그 이름을 주어로.
- ❌ 앤쓰로픽이 자기 모델로 보안 구멍을 수만 개 찾았습니다
- ✅ 앤쓰로픽 미토스가 보안 취약점 수만 개를 발견했습니다

**(2) 보도 기조의 전문어를 따라간다.** tier-1 매체 영역별 전문어 그대로.
- 보안: 보안 구멍 → 보안 취약점
- 법률: 법정 다툼 → 소송 / 합의. **소송을 건다는 "고소했습니다"로** (제소보다 고소를 쓴다)
- 상장: 비밀 제출 → **비공개 제출** (confidential filing은 "비공개로 제출")
- 인사: 사람을 자른다 → 감원

**(3) 구체적 동사로.**
- 했다 → 발표했다 / 공시했다 / 밝혔다
- 만들었다 → 공개했다 / 출시했다 / 내놓았다
- 찾았다 → 발견했다 / 적발했다
- 풀었다 → 공개했다 / 개방했다 / 확대했다
- 맺었다 → 체결했다 / 합의했다
- 줬다 → 지급했다 / 합의금을 물었다

**(4) 목적어를 살려라.** "수만 개 찾았다"보다 "보안 취약점 수만 개를 발견했다"가 한 줄로 와닿는다.

**(5) 불필요한 수식어를 뺀다.** 사건 자체로 무게가 충분하면 강조 부사·형용사를 들이밀지 않는다.
- 처음으로 / 첫 / 결국 / 마침내 / 또다시 / 전격 / 공식적으로 / 충격 / 놀라운 / 무려 / 자그마치
- 사실이어도 본문에 두고 제목에선 빼면 깔끔.

**(6) 제목에 날짜를 붙이지 않는다.** 제목 오른쪽에 "(월 일)" 같은 날짜 표기를 두지 않는다. 날짜는 본문 첫 문장에 녹인다.
- ❌ 앤쓰로픽이 클로드 오퍼스 4.8을 공개했습니다 (5월 28일)
- ✅ 앤쓰로픽이 클로드 오퍼스 4.8을 공개했습니다

## 8. 직접 인용 번역: 영어 구문 베끼지 말고 한국어 발화로 재현

영어 매체 인용을 한국어로 옮길 때 가장 자주 발생하는 흠. AI는 직접 인용을 "원문 충실"하게 옮겨야 한다고 착각해 영어 구문을 단어 단위로 베껴 따옴표 안만 진하게 번역체가 된다.

**핵심 원칙: 인용은 발화의 재현이지 문장의 번역이 아니다.** 그 인물이 한국어 화자였다면 같은 상황에서 어떻게 말했을까를 먼저 상상하고, 그 다음 영어 원문과 의미가 맞는지 검증한다. 거꾸로 하면 매번 번역체.

### 자주 나오는 영어 구문별 처치

**(1) "There is no X (in Y)" 패턴: 능동·구체로**
- ❌ "옵트아웃 옵션은 회사 노트북에 없습니다." (직역)
- ✅ "회사 노트북에서는 옵트아웃이 안 됩니다." (한국어 화자 발화)

**(2) 수동태 → 능동태**
- ❌ "이 데이터는 매우 엄격히 통제됩니다." (수동 직역)
- ✅ "이 데이터는 아주 엄격하게 관리하고 있습니다." (능동)

**(3) 영어 filler 직역 ("or anything like that" 등)**
- ❌ "이 데이터는 감시도, 성과 추적도, 그런 종류의 어떤 것도 아닙니다."
- ✅ "감시도 아니고, 성과 추적도 아닙니다. 그런 식으로 쓰는 데이터가 아예 아닙니다."

**(4) Sino-Korean 명사 chain → 동사로 풀기**
- ❌ "회사의 미래 최적 규모가 어느 정도일지 우리도 잘 모릅니다."
- ✅ "앞으로 회사가 어느 정도 규모여야 할지 우리도 아직 잘 모르겠습니다."

**(5) "how X do Y" 의문문 어순**
- ❌ "똑똑한 사람들이 컴퓨터로 일을 어떻게 처리하는지" (어색한 어순)
- ✅ "똑똑한 사람들이 컴퓨터로 어떻게 일하는지"

**(6) 영어 hedge·filler 위치**
- 한국어는 hedge를 어미로 해소: "~일 겁니다 / ~지 않을까 싶습니다 / ~이지 싶습니다"
- 영어 "probably" 자리를 무조건 따라가지 않는다.

**(7) 영어 한 문장 = 한국어 한 문장 강박**
- ❌ "거의 한 달의 모호함이 모두에게 남는다는 것, 매우 불안하실 거라는 걸 알고 있습니다."
- ✅ "거의 한 달간 답을 모르는 채로 기다려야 합니다. 모두에게 얼마나 무거울지, 잘 알고 있습니다."

### 화자 톤 캐스팅: 인용 번역 전 1초 체크

| 화자 유형 | 참조할 한국어 톤 |
|---|---|
| HR 메모·인사 책임자 | 한국 회사 인사팀 메모 (정중·약간 따뜻) |
| CEO·임원 공식 발언 | 한국 임원 컨퍼런스 콜·기자간담회 |
| 직원 익명 코멘트 | 한국 직장인 익명 게시판·블라인드 |
| 변호사·법률 문서 | 한국 법조계 변호인 의견서 |
| SNS·트윗 | 한국 X·인스타 캡션 |
| CTO·기술 임원 답변 | 한국 IT 기업 임원 사내 답변 (직설·간결) |

이 톤을 먼저 머리에 떠올린 다음 인용을 옮긴다. "한국 HR 책임자라면 이렇게 말 안 한다"는 신호가 오면 다시 쓴다.

### 인용 의미 명확성

직원이 "어떻게 옵트아웃하나요?"라고 묻고 CTO가 답하는 흐름이라면, 답이 그 흐름에 자연스럽게 맞물려야 한다.

- ❌ "회사 노트북에서는 옵트아웃 자체가 불가능합니다." ("자체가" 군더더기 강조, 답인지 사고문인지 흐림)
- ✅ "회사 노트북에서는 옵트아웃이 안 됩니다." (직접 답변 톤)

**원칙**: 군더더기 강조 어휘("자체가", "그 자체로", "원천적으로")는 의미가 흐려질 때 빼는 쪽이 자연스럽다.

### 인용 단위 검수 체크리스트

각 직접 인용 옮긴 뒤 자문:
- 한국어 화자가 같은 상황에서 이렇게 말할까?
- 영어 수동태를 한국어 피동으로 1:1 옮기지 않았나?
- "그런 종류의 어떤 것도", "관련해서는", "에 의해" 같은 직역 표현이 있나?
- 한자어 명사 chain을 동사로 풀 자리는 없나?
- 한 문장이 너무 길어서 두 문장으로 쪼갤 자리는 없나?
- 군더더기 강조("자체가", "원천적으로")가 의미를 흐리지 않나?

하나라도 ✓이면 다시 쓴다. 의미·사실은 보존하되 발화는 재현.

## 9. 의미 정확성 검수: 글 전체

### 주어·목적어가 의미상 맞는가

영어 직역에서 자주 발생하는 의미 어긋남. 한국어로 다시 읽었을 때 주어가 정확한 행위자인지 검증.

- ❌ "직원들이 일을 덜 하라고 만든 도구" (도구를 직원들이 만든 것처럼 읽힘)
- ✅ "사람의 일을 덜기 위해 만든 도구" (도구의 목적이 분명)

**원칙**: 한국어로 읽은 뒤 주어가 행위자인지, 의도된 의미인지 한 번 더 확인.

### 정권·시기가 다른 주체를 뭉뚱그리지 않는다

여러 해에 걸친 사건에서 "정부·회사·기관"을 한 덩어리로 묶으면 사실이 틀어진다. 특히 한국은 정권이 바뀌면 여당이 보수↔진보로 뒤집힌다. 2014년에 한 일과 2026년에 한 일을 "그 정부"로 묶으면 동일 주체가 한 것처럼 읽힌다.

- ❌ "카카오톡을 사찰한 그 정부가 이제 소버린 AI를 외칩니다." ("그" = 동일 정부로 오독. 실제론 박근혜 정부 vs 이재명 정부)
- ✅ "2014년 카카오톡을 사찰한 것이 정부였습니다. 12년 뒤 소버린 AI에 10조 원을 건 것도 정부입니다." (연도·시차로 분리해 제도로서의 '정부'를 가리킴)

**원칙**: 정권·경영진·기관장이 바뀐 사안은 시차(연도)를 명시해 분리한다. 정파를 굳이 본문에 박지 않더라도, "그 정부"처럼 동일 주체로 읽히는 지시어는 피한다. 역설은 "제도로서의 정부가 시기마다 정반대로 움직였다"로 충분히 산다.

### 중복 의미 표현 제거

같은 사실·감정을 두 번 표현하면 호흡이 빠지고 글이 늘어진다.

- ❌ "어떤 이들은 차라리 잘라줬으면 좋겠다는 심정을 토로하기도 했습니다." + "...일부는 사내에서 '잘려도 좋다'는 신호를 의도적으로 보내고 있다고 답했습니다." (두 문장이 같은 의미)
- ✅ NYT 인용 한 번만 + "퇴직금이라도 두둑이 받을 수만 있다면 말입니다" punchline

**원칙**: 두 문장이 거의 같은 뜻이면 더 구체적·인용 기반인 쪽을 남기고 다른 쪽은 뺀다.

### 본문 punchline과 끝줄 중복 피하기

본문 중간에 등장한 punchline 표현이 끝줄에 거의 그대로 다시 나오면 마지막 펀치가 무력해진다.

- ❌ 본문: "그 클릭은 모두 AI 훈련 데이터로 들어갑니다."
       끝줄: "그날 잘린 사람의 마지막 마우스 클릭도 AI 훈련 데이터로 수집될 것입니다." (중복)
- ✅ 본문: "그 클릭은 모두 AI 훈련 데이터로 들어갑니다."
       끝줄: "그날 잘린 사람의 마지막 마우스 클릭도 AI를 가르치는 자료로 남을 것입니다." (다른 표현, 콜백)

**원칙**: 끝줄은 본문 핵심 이미지와 호응하되 같은 어휘를 다시 쓰지 않는다. 다른 각도(자료/데이터셋/훈련용 입력)로 변주.

### 접속사 어색한 시작: "하지만,"

앞 문장과 contrast가 약하면 "하지만,"으로 시작하지 않는다. 부자연스럽다.

- ❌ "...사용량 경쟁이었습니다.\n\n하지만, 너무 많은 에이전트가 만들어진 나머지..." (앞뒤 contrast 약함)
- ✅ "...사용량 경쟁이었습니다.\n\n그러자 묘한 풍경이 펼쳐졌습니다. 너무 많은 에이전트가 만들어진 나머지..."

**원칙**: 다음 문단의 출발은 앞 문단과의 관계로 결정. contrast/순행/병렬 중 어느 것인지 보고 적절한 연결어 선택. 어울리는 게 없으면 연결어 빼고 새 문단으로.

### 문장과 문장은 흐름으로 잇는다

한 문단이 사실을 툭툭 나열하면 AI 보고서 냄새가 난다. 각 문장이 앞 문장을 받아 이어지게 가벼운 연결을 넣는다.

- ❌ 끊김: "...사전 승인을 받아야 합니다. 그간 핵 과학자에게만 적용하던 통제를 민간 인재로 넓혔습니다." (두 사실이 따로 논다)
- ✅ 이음: "이들은 이제 사전 승인을 받아야 합니다. 그간 핵 과학자에게만 적용하던 통제를 민간 인재에게까지 넓힌 것입니다."

**연결 도구**: 이제 / 앞서 / 그 무렵 / 여기에 / 한편 / 그렇게 (시간·대비·추가 관계에 맞춰).

**과잉 금지**: 모든 문장을 연결어로 묶지 않는다. §1.5 짧은 문장·긴 문장 섞기를 유지하고, 한 문단에서 같은 연결어를 반복하지 않는다.

## 10. 외래 조직·기술 용어 처리

한국 독자에게 즉시 와닿지 않는 외래 조직·기술 용어가 본문에 등장하면 영문 병기 + 짧은 설명이 필요하다. 그대로 두면 의미가 흐려진다.

- ❌ "회사를 'AI 포드'로 재조직하는 중입니다." (포드 자동차로 오해 가능)
- ✅ "회사를 'AI 포드(pod)'라는 소규모 자율 팀 단위로 재조직하는 중입니다."

**원칙**:
- 한국 매체에서 일반화되지 않은 외래 조직·기술 용어 (pod, cell, squad, OKR, sprint 등)는 첫 등장 시 영문 병기 + 한국어 짧은 설명
- 일반화된 용어 (CEO, AI, API, M&A 등)는 영문 병기 불필요
- 헷갈릴 수 있는 음차 표기 (포드 → 자동차 회사 Ford, 셀 → 휴대전화 cell, 스쿼드 → 군사 squad)는 반드시 병기·설명

**예외 면책**: 같은 본문 안에서 두 번째 등장부터는 병기 없이 사용 OK.

## 11. 숫자와 비율 함께 표기

큰 인원·금액 수치가 나오면 전체 대비 비율도 함께 보여 임팩트를 살린다. 숫자 단독은 규모감이 흐릴 수 있다.

- ❌ "메타는 5월 20일에 8천 명을 해고할 예정입니다."
- ✅ "메타는 5월 20일에 약 10퍼센트인 8천 명을 해고할 예정입니다." (전체 대비 비율 추가)

**원칙**:
- 인원·해고·매출·자본지출 등 큰 숫자는 가능한 한 분모(전체)와 함께
- 비율과 숫자 둘 다 보일 때 punchier
- 분모가 본문 다른 곳에 이미 나왔어도 같은 단락에 한 번 더 (예: "7만 8천 명 중 8천 명")
- 비율 표기는 한글 단위 ("10퍼센트", % 안 씀)

## 13. 일한 번역투 (일본어 직역)

AI가 쓴 한국어는 대개 영어 직역이지만, 한국어와 일본어는 어순(SOV)이 같고 한자어를 공유해 일본어 간섭도 함께 스며든다(김한식 2012, 오경순 2010). 아래 표현은 일본어 구문·조어를 그대로 옮긴 자리다. 한국어 문장으로 되돌려 쓴다.

| ❌ 일본어 직역 | 원 일본어 | ✅ 자연스러움 |
|---|---|---|
| ~에 다름 아니다 | にほかならない | 바로 ~다 / ~인 셈이다 |
| ~에 있어서 | における / において | ~에서 / ~할 때 / (대개 삭제) |
| ~하지 않으면 안 된다 | なければならない | ~해야 한다 |
| ~을 요하다 | を要する | ~이 필요하다 |
| ~에 값하다 | に値する | ~할 만하다 |
| 「 」 (낫표) | 「 」 | " " / ' ' (한국어 따옴표) |

**`~적(的)` 남용.** 일본식 조어 `~的`이 한 단락에 몰리면 번역투 신호입니다. `구조적·정책적`처럼 이미 정착된 말은 그대로 두되, `본질적으로·기본적으로·전면적으로`가 잇따르면 부사나 동사로 풉니다. "전면적으로 재검토했다" → "전면 재검토했다", "기본적으로 동의한다" → "큰 틀에서 동의한다". 과잉 검출이 잦아서 자동 감점하지 않고 눈으로만 봅니다.

**동형이의 한자어.** 같은 한자어라도 한일 간 뜻이 다른 경우를 조심합니다(김한식 2012). 일본어 `生鮮`은 고기·야채까지 포함한 신선식품을 뜻하고, `失踪`은 사건 보도에서 `行方不明(행방불명)`으로 씁니다. 원문이 일본어일 때 한자어를 그대로 음차하지 않았는지 살핍니다.

## 14. 담화 층위: 텍스트를 밑에서 받치는 흐름

번역투는 어휘와 문장에만 있지 않습니다. 접속과 지시로 문장을 잇는 응결 장치, 주제가 이어지는 방식, 문장부호처럼 글 전체를 받치는 담화 층위에도 번역의 흔적이 남습니다(김한식 2012, 김혜영 2009). 이 층위는 규칙으로 잡기 어려워 눈으로 봅니다.

### 응결 장치를 1:1로 옮기지 않는다

영어와 일본어는 접속사와 지시어를 자주 쓰지만, 한국어는 생략으로 잇습니다. 원문의 `and·but·그리고·그러나`를 문장마다 그대로 옮기면 접속어가 넘칩니다.

- ❌ "회의가 열렸다. 그리고 안건이 상정됐다. 그러나 표결은 미뤄졌다. 그래서 다음 주에 다시 모인다."
- ✅ "회의가 열려 안건을 상정했지만 표결은 미뤄졌다. 다음 주에 다시 모인다."

김혜영(2009)은 번역문에서 대조 접속 `그러나·하지만`이 잦고 전환 `그런데`가 드물다고 보고했습니다. 접속어가 한쪽으로 쏠리면 글이 기계적으로 읽힙니다. 관계에 맞는 접속어를 고르고, 어울리는 게 없으면 접속어를 빼고 잇습니다.

### 지시어·대명사를 덜어낸다

`이러한·그러한·이와 같은`은 영어 `such/this`와 일본어 `このような`의 직역 자리에서 늘어납니다. `그것·그들·그녀`도 마찬가지입니다(§1.10). 앞 문장을 받을 때 지시어를 반복하지 말고 생략하거나 구체 명사로 되받습니다.

- ❌ "이러한 결정은 이러한 배경에서 나왔다. 그리고 그것은 그들에게 큰 영향을 미쳤다."
- ✅ "이 결정은 그런 배경에서 나왔고, 직원들에게 큰 영향을 미쳤다."

### 주제를 이어 문단을 엮는다

영어는 새 문장마다 주어를 세우지만, 한국어는 앞 문장의 제재를 주제(`은/는`)로 받아 잇습니다. 문단이 매 문장 새 주어로 시작하면 사실이 툭툭 끊깁니다(§9의 "문장과 문장은 흐름으로 잇는다"와 짝).

- ❌ "정부가 정책을 발표했다. 기업이 반발했다. 시민이 우려했다." (주어가 매번 바뀌어 흐름이 끊김)
- ✅ "정부가 정책을 발표하자 기업이 곧바로 반발했다. 지켜보던 시민들도 우려를 내비쳤다."

### 문장부호도 담화를 받친다

쉼표와 따옴표는 글을 밑에서 받치는 담화 요소입니다(김한식 2012). 일본어는 주제어 `는/도` 뒤와 문두 접속사·부사 뒤에 쉼표를 찍어 문장당 쉼표가 한국어의 약 4.14배입니다. 이를 그대로 옮기면 쉼표가 넘칩니다.

- ❌ "그러나, 정부는, 어제, 새 방침을 발표했다."
- ✅ "그러나 정부는 어제 새 방침을 발표했다."

문두 접속부사 뒤(`그러나,`)와 주제어 뒤(`정부는,`) 쉼표는 빼는 게 자연스럽습니다. 일본어 낫표 `「 」`는 한국어 따옴표로 바꿉니다.
