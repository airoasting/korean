# 자동 fail / 경계 그룹 / 금지 사항

9.7군 강도. 한 항이라도 걸리면 즉시 fail 처리.

## 자동 fail 그룹

| 항목 | 흔한 실패 문구 |
|---|---|
| em dash 본문 | `—`, `–` |
| 번역투 치명 잔존 | `~를 통해`, `~에 대해`, `~에 기반하여`, `~함에 있어` |
| AI 상투구 | `결론적으로`, `시사하는 바가 크다`, `~할 필요가 있다`, `우리에게 던지는 화두`, `잊지 말아야 한다` |
| 과장 어휘 | `혁신적`, `획기적`, `중대한`, `심오한`, `놀라운`, `뜻깊은` |
| 이중 피동 | `~되어진다`, `~지게 된다`, `~되어지고 있다` |
| 추측 어미 | `~인 것 같다`, `~인 듯하다`, `~로 보인다`, `~라고 할 수 있다` |
| 메타 표현 | `이 글에서는`, `정리하자면`, `다시 말해`, `앞서 말했듯이` |
| 어색한 동사구 (#11) | `달러를 향했다`, `로 흘러갔다`, `에 다가가고 있다`, `를 마주했다` |
| AI 마무리 명언 (#12) | `자기 길을 찾는 중`, `시대가 왔습니다`, `시간이 시작됐습니다`, `신호입니다`, `선이 그어지는 자리`, `기로에 섰습니다`, `~한 셈입니다` (추상 일반화) |
| 식상한 비유 (#13) | `법정에 섰다`, `본진이 나섰다`, `수면 위로 떠올랐다`, `포문을 열었다`, `닻을 올렸다`, `기지개를 켰다`, `춘추전국시대`, `신호탄을 쏘아 올렸다` |
| 인용 동사 generic 반복 (#14) | `~다고 했습니다`/`라고 했습니다`가 한 산출물에 3회 이상 |
| 의미 변경 | 사실·숫자·고유명사·인용 한 글자라도 바뀜 |
| 다리 놓기 (지어내기) | 파편·비어 있는 원문을 메우려 원문에 없는 의미·연결·해석을 창작. 예: "신호입니다. 시대가 왔습니다." → "그 신뢰가 흔들립니다"처럼 없던 인과를 지어냄 |
| 말투 이탈 | 격식체 입력 → 비격식체 출력 |
| 변경률 폭주 | 50% 강제 중단을 무시하고 계속 다듬기 |
| 페르소나 정체성 침범 | RED가 분야 디테일·감정으로, SILVER가 일반론·감정으로, BLUE가 논리 분석으로 흐름. GOLD가 분석 어휘 사용 |

> em dash 금지의 근거: 김정우(2011a)의 영한 번역문 조사에서 영어 원문의 줄표(dash)가 한국어 번역문에서 동일한 줄표로 옮겨진 용례가 **단 한 건도 없었다**. 전문 번역자·편집자 집단이 줄표를 통상적 한국어 문장 부호로 받아들이지 않는다는 뜻(번역 보편소 중 규범화). 마침표·쉼표·괄호·콜론으로 옮긴다.

### 두 가지 단서

- **추측 어미의 장르 면책.** `~인지 모른다`, `~일지 모른다`, `~지 않을까`가 **칼럼·에세이에서 화자의 태도**로 쓰인 자리는 자동 fail에서 면책한다(genre-rules의 "칼럼은 화자 어조 보존"이 우선). 반대로 **뉴스·리포트**에서 사실을 흐리는 추측(`~로 보인다`, `~인 것으로 보인다`)은 그대로 fail이다. 판정이 갈리면 장르를 먼저 확정하고, 화자의 의견인지 사실 서술인지로 가른다.
- **마무리 명언·비유는 종결체 변이형까지 fail.** 표에는 `시대가 왔습니다`처럼 격식체만 적었지만, 해라체 `시대가 왔다`, `~온 것이다`, `~왔어요`도 같은 자동 fail이다. `count-patterns.py`는 어미를 자른 stem으로 잡는다. 특정 어미형만 피해 가도 통과되지 않는다.

## 경계 그룹 (자동 fail 아님, 비평가가 짚는 자리)

- 경고 잔존 빈도. 1~2개는 허용, 3개 이상이면 SILVER가 짚는다.
- 형용사 과잉. 명사 한 개에 장식 형용사가 둘 이상 붙으면 SILVER가 짚는다.
- 단락 첫머리에 접속사 의존 (`또한`, `그러나`, `한편`, `더불어`, `따라서`).
- 같은 길이 문장 다섯 줄 연속.

## AI 글 다듬기 특화 금지

- **영어 병기 남발 0회.** 원문에 없으면 추가하지 않는다 (`혁신(innovation)`, `통찰(insight)` 같은 자리).
  - **면책 조항**: 회사명·제품명·인명·이벤트명 같은 **고유명사의 첫 등장 시 영어 병기 1회**는 허용. 두 번째 등장부터는 한국어 표기로 통일하면 OK. 일반 명사("혁신·통찰·전략")는 그대로 0회 원칙.
- **단락당 이모지 1회 이하.**
- **본문 80% 이상이 글머리 기호인 형식 금지.**
- **원문에 없는 사실·숫자·고유명사·인용 추가 0회.** 의미 보존 위반.

## "의미 동등"의 허용 폭

다음은 의미 변경이 아니다 (다듬기 가능):
- 어순 재배치
- 같은 뜻 어휘 교체 (`이루어졌다` → `벌어졌다`, `~를 통해` → `~로`)
- 길어진 절을 두 문장으로 쪼개기
- 능동/피동 전환 (행위자가 분명할 때)
- 한자어를 더 자연스러운 한국어로 (`수행하다` → `한다`, `진행하다` → `한다`)

다음은 의미 변경이다 (자동 fail):
- 사실 한 글자라도 바꿈
- 숫자·날짜·통계값 변경
- 고유명사(회사명·인명·제품명·지명) 변경
- 직접 인용("...") 안의 글자 변경
- 원문에 없는 정보 추가
- 원문에 있는 핵심 정보 삭제
- 어조의 강도 임의 조정 ("주장했다" ↔ "말했다" 임의 교체. 원문이 said/claimed/argued 중 어느 쪽이었는지 따라감)
